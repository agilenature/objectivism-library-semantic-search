## Algorithm Implementation

### Pass 1 — Genus Identification (pure DB computation)
- Series: ITOE Advanced Topics (58 files), Objectivist Logic (50 files)
- ITOE AT: 490 unique aspects, 0 reach 80% threshold → all aspects are differentia
- OL: 432 unique aspects, 0 reach 80% threshold → same
- Genus subtraction step: vacuous for both series (no shared genus at 80% threshold)
- Algorithm degrades gracefully to pure rarity discrimination when genus is empty

### Pass 2 — Differential Identification (Claude Code session judgment)
Inline philosophical discrimination applied within this session, bypassing Anthropic API.
The `phrase_override` parameter added to `get_claude_discrimination_phrase()` allows
inline judgment without any API dependency.

**Key insight confirmed:** When 240/490 ITOE AT aspects all tie at series freq=1,
frequency sorting alone cannot discriminate. "Zeno's arrow paradox" and "order of being
vs. order of knowing" both score [1/58, 1 corpus] — identical under any frequency metric.
Philosophical specificity (names a concrete historical problem vs. describes an abstract
relation) is the correct discriminating criterion, requiring judgment.

### Pass 3 — Essentialization
Modified to use `discrimination_phrase` directly when it fits within the 7-word budget,
rather than rebuilding from aspect strings (which may exceed the budget when concatenated).
This ensures the phrase from Pass 2 reaches validation unchanged.

## Implementation Decisions

1. **phrase_override parameter**: `get_claude_discrimination_phrase()` accepts an optional
   `phrase_override` dict that bypasses API entirely. The running Claude Code session IS
   the discriminating agent. This pattern handles cases where ANTHROPIC_API_KEY is not set.

2. **Pass 3 direct phrase**: `build_discrimination_phrase()` now uses `discrimination_phrase`
   directly when provided and within word budget, instead of concatenating aspect strings
   (which fails when individual aspects already use the full 7-word budget).

3. **Gemini store key**: DB uses `gemini_store_name` (not `gemini_store_resource_name`).
   `crad_pilot.py:setup_gemini()` falls back to API listing when key not found — this works
   but is slower. The correct key is verified.
