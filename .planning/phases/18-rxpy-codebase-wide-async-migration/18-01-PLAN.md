**Pattern 3: dynamic_semaphore(limit$: BehaviorSubject[int]) operator**

**Pattern 4: Two-signal shutdown via shutdown_gate operator**

Ctrl-C / emergency: fire both simultaneously.
Three sub-tests required:
  (1) fire stop_accepting$ -> in-flight items complete, no new items accepted
  (2) fire force_kill$ -> immediate termination of active chains
  (3) normal shutdown sequence: stop_accepting$ -> drain -> force_kill$

**Pattern 5: Tenacity AsyncRetrying -> RxPY retry_when**

```python
class OCCConflictError(Exception):
    pass

def occ_transition(fn, max_attempts=5, base_delay=0.1):
    """Retry fn() on OCCConflictError with exponential backoff + jitter."""
    import random

    def subscribe(observer, scheduler=None):
        async def run():
            for attempt in range(max_attempts):
                try:
                    result = await fn()
                    observer.on_next(result)
                    observer.on_completed()
                    return
                except OCCConflictError:
                    if attempt == max_attempts - 1:
                        observer.on_error(OCCConflictError(f"OCC conflict after {max_attempts} attempts"))
                        return
                    delay = min(base_delay * (2 ** attempt) + random.random() * 0.05, 1.0)
                    await asyncio.sleep(delay)
        asyncio.ensure_future(run())

    return rx.create(subscribe)
```

```python
def shutdown_gate(obs, stop_accepting$, force_kill$=None):
    """Two-signal shutdown operator.

    stop_accepting$: gates input — no new items enter when fired
    force_kill$: take_until on active chains — immediate termination
    """
    gated = obs.pipe(ops.take_until(stop_accepting$))
```

terminates even in-flight items. These are semantically different — `stop_accepting$`

```python
def make_retrying_observable(fn, max_retries=5, base_delay=1.0):
    attempt_count = [0]

    def subscribe(observer, scheduler=None):
        async def run():
            try:
                result = await fn()
                observer.on_next(result)
                observer.on_completed()
            except Exception as e:
                observer.on_error(e)
        asyncio.ensure_future(run())

    return rx.create(subscribe).pipe(
        ops.retry_when(
            lambda errors: errors.pipe(
                ops.scan(lambda acc, e: acc + 1, 0),
                ops.flat_map(
                    lambda n: rx.throw(Exception("max retries")) if n > max_retries
                    else rx.timer(base_delay * (2 ** (n - 1)))
                )
            )
        )
    )
```

**Section 4: Production Implementation Guidelines**
For each tier:
- Tier 3: `asyncio.to_thread` replacement — do NOT use `.run()` inside async service methods.
  The correct pattern is the Future-based subscription:
  ```python
  result_future = asyncio.get_event_loop().create_future()
  rx.from_callable(lambda: fn(*args), scheduler=NewThreadScheduler()).subscribe(
      on_next=lambda v: result_future.set_result(v),
      on_error=lambda e: result_future.set_exception(e),
      on_completed=lambda: None,
  )
  result = await result_future
  ```
  `.run()` blocks the calling thread via `threading.Event.wait()`. Inside `async def`,
  this permanently blocks the asyncio event loop — coroutines scheduled with
  `asyncio.ensure_future()` never execute. `.run()` must only be used at the top-level
  (outside any running event loop).
- Tier 2: `asyncio.Semaphore` + `gather` -> `ops.map(factory).pipe(ops.merge(max_concurrent=N))`.
  NOTE: Use `ops.map(...).pipe(ops.merge(max_concurrent=N))` — NOT `ops.flat_map(mapper, max_concurrent=N)`.
  RxPY 3.x `ops.flat_map()` does not accept `max_concurrent`; `ops.merge()` is the correct
  operator for bounded concurrency. `AsyncLimiter` -> `throttle_with_timeout`.
  Async coroutines inside the pipeline must use `asyncio.create_task(coro)` + `rx.from_future()`,
  NOT `asyncio.run(coro)` (which raises RuntimeError when an event loop is already running).
- Tier 1: All custom operators from Section 3; dynamic_semaphore approach (from Pattern 3);
  two-signal shutdown_gate (from Pattern 4); `await occ_transition_async(...)` in state.py
  async methods (NOT `.run()`).
